EOS in a Box, draft 0.01

Will have simple jQuery input fields and simple interaction, no sockets, no templates, no react, no nothing. The focus is to make webauthn work in a minimal setup.

The front end will have a field to enter a new account name, the user then should click a button to check availability and the server should check with several API endpoints if it is available to create a new account on the blockchain.

in the near future there will be a config.json file in the back end where the site owner can enter config info. The starting template will be self-explanatory. Things like the website owner name which will be presented in to the user and a paragraph which the site owner can modify. There will be a footer line which will be "hard coded" in the html.

imporant items in the config will be, the list of blockchains this minisite supports e.g. EOS, jungle3, WAX, TELOS, PROTON... with the "human readable blockchain name", the actual blockchain hash identifier and a list of api endpoints for this blockchain
